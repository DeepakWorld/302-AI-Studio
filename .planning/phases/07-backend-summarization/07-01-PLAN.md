**1. Add /generate-context-summary POST endpoint to router.ts**

Insert the new endpoint AFTER the /generate-title endpoint (after line ~1088) and BEFORE /generate-suggestions.

The endpoint structure mirrors /generate-title exactly. Copy the pattern:

Request body:
```typescript
{
    messages: UIMessage[];
    model: string;
    apiKey?: string;
    baseUrl?: string;
    providerType: ModelProvider["apiType"];
    previousSummary?: string;
    language?: string;
}
```

Response: `{ summary: string }` on success, `{ error: string }` with 400/500 on failure.

Implementation steps (all copied from /generate-title):
1. Parse request body with destructuring
2. Convert messages to conversation text using the SAME message-to-text logic (filter text parts, format as "User: ...\nAssistant: ...")
3. Create languageModel using the SAME provider switch (302ai uses createOpenAICompatible, openai uses createOpenAI, anthropic uses createAnthropic, gemini uses createGoogleGenerativeAI). Copy the switch block verbatim from /generate-title.
4. Build prompt based on whether previousSummary exists:

**First generation (no previousSummary):**
```
Summarize the following conversation into a concise context summary (200-500 characters). Preserve:
- Key topics and decisions made
- Important facts, names, numbers, and dates mentioned
- User preferences and requirements stated
- Any ongoing tasks or action items

Conversation:
${conversationText}

Requirements:
- Use the SAME LANGUAGE as the main language in the conversation
- Keep between 200-500 characters
- Focus on information that would be useful context for continuing the conversation
- Do not include pleasantries or greetings

Return ONLY a valid JSON object in this exact format (no markdown, no code blocks):
{"summary": "your summary here"}
```

**Incremental update (has previousSummary):**
```
Update the following context summary with new conversation messages. The updated summary should be 200-500 characters and incorporate the new information while preserving important context from the previous summary.

Previous Summary: ${previousSummary}

New Messages:
${conversationText}

Requirements:
- Use the SAME LANGUAGE as the main language in the conversation
- Keep between 200-500 characters
- Preserve key facts, decisions, and context from the previous summary
- Add new important information from the latest messages
- Remove outdated or superseded information
- Focus on information useful for continuing the conversation

Return ONLY a valid JSON object in this exact format (no markdown, no code blocks):
{"summary": "your updated summary here"}
```

5. Call `generateText({ model: languageModel, prompt })` -- NOT streamText
6. Parse the response using the SAME JSON parsing logic from /generate-title:
   - Strip thinking/reasoning blocks (3 patterns: closed tags, unclosed-before-JSON, remaining-opening)
   - Remove markdown code blocks
   - JSON.parse to extract `parsed.summary`
7. On JSON parse failure, use fallback: strip thinking tags from raw text, truncate to 500 chars
8. On success: `return c.json({ summary })`
9. On error: `return c.json({ error: "Failed to generate context summary" }, 500)`

**2. Create src/lib/api/context-summary-generation.ts**

Mirror src/lib/api/title-generation.ts structure exactly. Import the SAME types used by title-generation.ts.

Define interfaces:
```typescript
export interface GenerateContextSummaryRequest {
    messages: ChatMessage[];
    model: string;
    apiKey?: string;
    baseUrl?: string;
    providerType: "302ai" | "openai" | "anthropic" | "gemini";
    previousSummary?: string;
    language?: string;
}

export interface GenerateContextSummaryResponse {
    summary: string;
}
```

Import `FallbackModelConfig` from `./title-generation` (reuse, do not duplicate).

Inner function `generateContextSummaryRequest()`:
- Parameters: messages, modelId, provider, serverPort, previousSummary, language, signal
- Calls `fetch("http://localhost:${serverPort}/generate-context-summary", { signal, method: "POST", ... })`
- Returns `string` (the summary text)
- Throws on non-ok response

Outer function `generateContextSummary()`:
- Parameters: messages, model (Model), provider, serverPort, previousSummary, language, fallbackConfig, signal
- Tries primary model via inner function
- On failure: checks abort, logs error, waits 500ms, determines fallback model (from fallbackConfig or hardcoded gpt-4o-mini)
- If fallback === primary, returns null
- Tries fallback via inner function
- On total failure: returns null (never throws except AbortError)
- Return type: `Promise<string | null>`

Do NOT include the `sanitizeGeneratedTitle` function -- that is title-specific. The summary response is just a string, no sanitization needed beyond what the server already does.

```typescript
/**
 * Cancel any pending context summary generation request.
 * Should be called before sending a new message to avoid race conditions.
 */
cancelPendingSummary() {
    if (this.summaryAbortController) {
        this.summaryAbortController.abort();
        this.summaryAbortController = null;
        console.log("[ContextSummary] Cancelled pending summary generation");
    }
}
```

**3. Add createSummaryAbortController method** (after createTitleAbortController, around line 255):
```typescript
/**
 * Create a new AbortController for context summary generation.
 * Returns the AbortSignal to be passed to the fetch request.
 */
createSummaryAbortController(): AbortSignal {
    // Cancel any existing pending request first
    this.cancelPendingSummary();
    this.summaryAbortController = new AbortController();
    return this.summaryAbortController.signal;
}
```

<task type="auto">
  <name>Task 2: Apply fade animation to streaming markdown blocks</name>
  <files>src/lib/components/buss/markdown/markdown-renderer.svelte</files>
  <action>
Modify the MarkdownRenderer to apply fade animation to blocks during streaming:

1. Track which blocks are "new" by comparing current blocks to previous render
2. Add the `streaming-fade-in` class to newly added blocks when `isStreaming` is true
3. Use a reactive approach: maintain a Set of "seen" block IDs, apply animation class to blocks not yet seen

Implementation approach:
- Add a `$state` variable `seenBlockIds = new Set<string>()` to track rendered blocks
- In the `$effect` that calls `collectBlocks`, after blocks are updated, identify new block IDs
- In the template, conditionally apply `streaming-fade-in` class when:
  - `props.isStreaming` is true AND
  - block.id is NOT in seenBlockIds (it's a new block)
- After rendering, add the block.id to seenBlockIds

Apply the animation to all block types: html, code, todo, write, tool-loading.

Important: Reset seenBlockIds when content is cleared or message changes (when content becomes empty or messageId changes).
  </action>

1. Run `pnpm run check` to verify no TypeScript errors
2. Run `pnpm run lint` to verify no linting issues
  </verify>

Purpose: Allow users to capture task ideas immediately without waiting for AI output to complete. When user types and presses Enter while AI is streaming in Vibe Mode, the input is added as a task to the taskboard instead of being queued as a chat message.

Output: Modified taskboard state with new method, updated chat-input-box with redirection logic, and i18n messages for toast notification.
</objective>

1. Add a derived state `shouldRedirectToTaskboard` near line 51 (after `shouldShowTaskboardStatus`):
   ```typescript
   const shouldRedirectToTaskboard = $derived(
     codeAgentState.enabled &&
     codeAgentState.inCodeAgentMode &&
     (chatState.isStreaming || chatState.isSubmitted)
   );
   ```
   This checks:
   - Vibe mode is enabled (`codeAgentState.enabled`)
   - Not a fresh tab, has messages (`codeAgentState.inCodeAgentMode`)
   - AI is currently streaming or submitted (`chatState.isStreaming || chatState.isSubmitted`)

2. Modify the `.otherwise()` branch in `handleSendMessage()` (around line 176-182):
   - BEFORE the existing `if (chatState.hasMessages)` block, add redirection check:
   ```typescript
   .otherwise(() => {
     // Redirect to taskboard if streaming in Vibe Mode
     if (shouldRedirectToTaskboard) {
       const content = chatState.inputValue.trim();
       if (content) {
         codeAgentTaskboardState.addTaskFromChatInput(content);
         toast.success(m.taskboard_task_added_from_chat());
         chatState.inputValue = "";
         chatState.attachments = [];
       }
       return;
     }

     // Original send logic
     if (chatState.hasMessages) {
       chatState.sendMessage();
     } else {
       document.startViewTransition(() => chatState.sendMessage());
     }
   });
   ```

3. Add import for the i18n message if not already imported (m is already imported from paraglide).

Key points:
- Check `shouldRedirectToTaskboard` FIRST in .otherwise() branch
- Clear BOTH `chatState.inputValue` AND `chatState.attachments` after adding task
- Use `toast.success()` for confirmation (matches existing patterns)
- Return early after redirection to prevent normal send

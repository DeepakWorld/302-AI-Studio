# Milestone v1.2: Auto Context Compression

**Status:** ✅ SHIPPED 2026-02-06
**Phases:** 6-9
**Total Plans:** 6

## Overview

This milestone enables automatic context compression for long conversations. When message count exceeds a configurable threshold, older messages are summarized into a rolling summary that gets injected as context when sending to AI. Original messages remain in UI -- compression is purely a transport optimization. Code Agent and private chat modes are exempt to preserve full context where needed.

## Milestone Summary

**Key Decisions:**
- Compression enabled by default, 20-message threshold, clamped 5-100
- Thread-level compressionEnabled allows per-thread override of global setting
- shouldApplyCompression exemption order: global > per-thread > Code Agent > private chat
- Reuse FallbackModelConfig from title-generation.ts for context summary API wrapper
- Language param accepted in API but prompt auto-detects from conversation
- Reuse titleGenerationModel for summary generation (no separate model setting)
- Summary generation awaited in onFinish (not fire-and-forget) for state consistency
- Compression applied AFTER template resolution, BEFORE convertToModelMessages
- Context summary prepended to system prompt with [Context from earlier conversation] markers
- FileStack icon for compression indicator, plain text summary (no Markdown)

**Issues Resolved:**
- None — clean implementation

**Issues Deferred:**
- Per-thread compression toggle in UI (future enhancement)
- Manual summary regeneration button (future enhancement)
- Pinned messages exempt from compression (future enhancement)

**Technical Debt Incurred:**
